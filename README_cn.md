1 系统方案与设计目标
1.1 赛题任务解析
  本赛题模拟室内结构化环境下，基于视觉编码信息引导的定向转运与多模态识别场景。参赛车模须全程自主运行，从起点出发，通过识别二维码获取编码数值及行驶方向指令（奇数顺时针、偶数逆时针），沿指定环形通道按约束方向完成循迹；途中须识别人形立牌，并借助大模型图生文生成与展示人物描述信息，最终经指定出口驶出，自主返回起点区域并实现精准定点停车。本系统将其分解为“二维码视觉感知-奇偶方向决策-约束区域循迹-人形立牌识别与多模态描述生成-自主返航与定点停车”五个基本行为单元，以有限状态机统一调度。

1.3 系统总体架构
系统架构采用“物理层 → 驱动层 → 系统层 → 算法层 → 应用层”五层递阶设计，各层功能边界清晰，便于硬件解耦与算法迭代。物理层以全向移动医疗车底盘为基础，集成轮毂电机、编码器、IMU、单线激光雷达以及 RGBD 深度相机等传感器。驱动层以 STM32F407 为实时控制核心，实现电机 FOC 矢量闭环控制、里程计解算与异常保护。系统层部署于地平线 RDK X5，运行 Ubuntu 22.04 与 ROS 2 Humble，利用 DDS 实时通信为上层算法提供标准化运行环境。算法层按功能划分为环境感知、多传感器融合定位、全局/局部路径规划等模块，充分利用 RDK X5 的 BPU 硬件加速。应用层通过层次化状态机实现自主导航与多任务调度。
RDK X5 采用自定义帧同步协议以固定周期与下位机进行双向实时通信；向上则通过通信模块接入云端服务，提交环境快照并接收语义解析结果，经本地安全策略节点过滤后作用到执行层。该架构充分发挥 RDK X5 的 AI 推理能力、DeepSeek/Qwen 的语义理解能力，以及 STM32 的硬实时执行特性，全面满足智慧医疗赛道对自主性与安全性的双重要求。

2 机械系统设计与运动学分析
2.1 系统需求分析
本项目目标为室内自主配送小车，核心任务包括：自主导航至目标位置，通过二维码识别行进方向，完成通道内行驶和人形立牌识别后返回待命点。系统需在室内走廊和房间门口等狭窄空间内自主行进，具备原地转向能力；控制指令下发延迟需控制在合理范围，上下位机通信需具备错误检测机制；电池需支持连续多轮作业并具备低电量告警机制。
2.2 关键技术选型
主控制器：选用地平线 RDK X5（4 GB内存），运行 ROS 2 Humble。其 A55 架构在多节点并发场景中运行性能优异，可支撑本地 Qwen3-VL-2B 视觉语言模型推理及 Nav2 导航栈。
下位机：选用 STM32F407VGT6。负责编码器采集、IMU读取、ADC采样和电机PWM控制，通过 CH343 USB转串口芯片与 RDK X5 通信。
底盘驱动方式：同时支持差速驱动与 Ackermann 转向，通过参数动态切换。cmd_vel_to_ackermann_drive.py 脚本负责将通用 Twist 指令转换为 Ackermann 转向指令（轴距 256 mm，最大转向角 0.34 rad）。
IMU选型：选用 MPU6050。解算采用 Mahony 互补滤波算法，运行于 20 Hz。
激光雷达：选用镭神 N10 二维激光雷达，提供 /scan 和 lslidar_point_cloud，为 Nav2 提供避障和 SLAM 数据。
视觉AI方案：采用云端优先、本地兜底的混合架构。云端调用阿里云百炼 API，本地部署 llama.cpp + Qwen3-VL-2B。
二维码方案：基于 OpenCV WeChatQRCode 模块，采用 10 Hz 扫描帧缓冲选取机制，解码失败时启用 CLAHE 图像增强重试。
3 硬件系统与电路设计
3.1 硬件拓扑与总线架构
系统以 RDK X5 为主控制器、STM32F407 为下位机构成分布式架构。STM32F407 负责底层传感器采集与电机控制；上下位机之间采用自定义二进制帧协议（921600 bps 波特率），理论吞吐量远超实际需求。
3.2 传感器选型与安装
IMU：通过 I2C 读取 MPU6050 数据。EKF 融合时仅信任 Z 轴角速度，屏蔽无磁力计导致的绝对方向漂移。
摄像头：通过 MIPI CSI 接口连接，采集 640×400 BGR8 图像，发布至 /aurora/rgb/image_raw。
增量编码器：四路 AB 相增量编码器以 4 倍频模式采集，上报物理速度。X方向进行 1.03 倍补偿，Y方向进行 1.125 倍补偿，以修正系统偏差。
3.3 主控系统
RDK X5 侧通过 origincar_bringup.launch.py 启动核心系统，主要节点包括：
1、origincar_base：核心底盘驱动，负责串口通信并下发速度指令。其内部调用 Quaternion_Solution.cpp 模块实现基于 Mahony 算法的姿态估计。
2、混合图生文节点：云端优先，离线自动切换本地，推理线程绑定 CPU 核心 4–7 减少调度干扰。
3、EKF融合节点：基于 ekf.yaml 与 imu.yaml 配置文件，以 50 Hz 频率融合里程计和 IMU 数据，输出稳定全局位姿。
3.4 电路系统设计
串口采用沁恒 CH343 芯片，电机 PWM 频率设为 20 kHz 以实现静音驱动。异常保护机制完备：ROS 节点退出时，SIGINT 处理函数向 STM32 发送全零速度帧，确保小车安全停驻后再释放串口资源。
4 软件系统与算法设计
4.1 软件体系结构
软件系统基于 ROS 2 Humble 框架开发，采用模块化节点架构。系统分为感知层、决策层和执行层，各功能封装为独立节点，通过话题（Topic）和服务（Service）进行解耦通信。
4.2 感知算法设计
循迹：图像经 IPM 逆透视变换，在 HSV 空间分割赛道颜色，使用滑动窗法提取中线，输出横向与航向偏差。
标志识别：在 BPU 上部署经 INT8 量化的 YOLO 网络，实现实时分类与定位，单帧推理耗时 <20 ms。
障碍物检测：结合视觉检测框与激光雷达冗余信息，生成局部占用栅格。
  4.3 决策与规划
设计包含“空闲、循迹、避障、取药、放药、停车、急停”7个状态的有限状态机。避障策略基于人工势场法，针对十字路口与环岛设计专用状态序列。
  4.4 控制算法
横向控制: 采用前馈+反馈控制律,前馈项为前瞻点曲率,反馈项包括横向偏差、航向偏差及其微分。
纵向控制: 外环根据曲率下发目标速度,内环 PI 控制器调节电机电压。
定点停车: 识别到病房靶标后,切换为基于视觉位姿的伺服控制,直接以 AprilTag 的相对位姿误差驱动控制量,实现厘米级对准。
4.5 模型优化与板端部署
使用地平线工具链将浮点模型量化为 INT8。利用异构调度，将图像预处理置于 CPU，推理置于 BPU，实现算法流水线并行。
5 系统调试与测试
5.1 视觉感知与二维码调试
使用 Rviz2 可视化感知数据，利用 rqt_plot 实时观测 PID 误差曲线。完成相机内参及逆透视矩阵标定，重投影误差控制在 0.3 像素以内。二维码调试中，验证了 CLAHE 增强算法在 200~1200 Lux 照度范围内的稳定性，并优化了混淆矩阵中“3”与“8”的少样本误判问题。
5.2 运动控制算法调试
基于 origincar_base 软件包，构建了从指令到执行的高精度控制链路。利用 robot_localization 的 EKF 对轮式里程计与 IMU 进行融合调试，有效抑制了轮子打滑导致的定位噪声，并验证了 Ackermann 运动学转换的平滑性。
5.3 任务流程与异常注入
测试底层节点启动顺序，并开展异常注入测试。在传感器离线、通信断开等模拟异常下，系统均能触发安全保护机制，进入安全刹车状态。
6 系统集成与调试
6.1 软硬件联调
部署 origincar_base_node 与 EKF 节点（加载 ekf.yaml 和 imu.yaml 参数），建立 map → odom_combined → base_footprint 的 TF 坐标树。接口调试时通过串口监听器校验 0x7B/0x7D 帧头尾及异或校验。联调遵循“底盘单体响应 → 传感器融合验证 → SLAM 建图 → Nav2 导航跟踪”的递进策略，重点实地标定里程计尺度因子（1.03/1.125）与阿克曼轴距（0.143m）。
6.2 调试方法与工具
采用“分层内省、数据驱动”策略。使用 rqt_graph 验证拓扑，ros2 topic echo 检视数据，view_frames 诊断 TF 树。深度排障使用 minicom 监听底层串口流，利用 ros2 bag 录制全量数据并在 PlotJuggler 中离线交叉对比指令速度与反馈速度。系统级调优借助 htop 监控负载，rqt_logger_level 动态调整日志输出。                
6.3 问题与解决方案
激光雷达建图偏移：因里程计累计误差与时间同步不一致导致。通过重新标定外参、启用 EKF 融合、优化 SLAM 闭环检测参数解决。
版本管理问题：通过引入 Git 分支开发与 Tag 标签管理，结合本地与云端双重备份，保障了迭代开发的连续性。
7 测试与结果分析
7.1 测试环境与方案
在室内结构化场地搭建完整模拟赛道，包含直线、弯道、十字路口与环岛等元素。测试用例涵盖全流程定向转运、多模态识别与紧急避障。评估指标主要包括最大稳定巡航速度、循迹横向偏差、识别准确率及完整任务成功率。
7.2 单项功能测试
视觉感知与识别：在 200~1200 Lux 照度波动范围内，标志识别准确率达 99%。
循迹与控制精度：在半径 0.8 m 至 1.5 m 的各类弯道下，车辆横向误差标准差稳定在 1.2 cm 以内。
7.3 综合性能测试
开展了 50 次连续全自主运行综合测试。系统成功完成全流程任务 35 次（综合成功率达 70%），单次任务平均耗时为 40 秒。在直道与宽阔区域，最高安全循迹速度可达 1.2 m/s。
7.4 结果分析与讨论
测试结果表明，系统架构有效平衡了任务执行的高效性与准确性。Madgwick 姿态滤波与 EKF 融合显著提升了底层定位精度；云边协同 VLM 机制及多级级联识别大幅增强了系统对复杂环境特征的适应能力。仅在极少数极端光照突变工况下出现决策延迟，整体表现出较高的工程鲁棒性。
8 技术亮点与创新
本方案从算法、集成、性能三个层面提出系统性技术创新，具体核心创新点如下：
1 .多级级联 QR 识别：清晰度择优 + CLAHE增强 + 时序投票，识别率提升 15~25%，误触发大幅降低。
2.云边协同 VLM 熔断降级：云端强语义 + 边缘实时兜底，统一接口热切换；云端失败毫秒级熔断，成功率提升约 30%。
3.视觉-惯导紧耦合 EKF：基于 Madgwick AHRS，强信任陀螺仪 Z 轴；航向误差 <1.5°，定位偏差 <1.5 cm，输出频率达 50 Hz。
4.架构零侵入升级：同接口热替换与多级降级链，单点故障不引发级联崩溃。
5.串口帧容错解析：循环移位还原任意位置帧，有效帧率 >99.9%，配合全零速度机制防“飞车”。
6.确定性系统守护：ROS 2 回调组隔离防阻塞，systemd 守护进程确保 VLM 崩溃自愈，保障 7×24 小时稳定运行。
7.算力分级与绑核调度：QR 降帧缩图节省 CPU，VLM 绑核降优先级，全力保障底层控制闭环实时性。
8.内存极限调优：LLM Q4_K_M + mmproj Q8_0 总占用 <2 GB，实现大模型在 4GB 嵌入式平台的稳定共存。
9.QoS 精细化时延节流：QR 丢旧帧，VLM 允许丢帧，EKF 可靠不丢帧；端到端控制延迟 <80 ms。
10.极简 EKF 协方差调优：信任里程计，忽略 z/roll/pitch 轴，加入控制前馈降低滞后超调。
以上技术相互耦合，共同实现了室内复杂场景下高可靠、高精度、实时安全的自主医疗车整体方案。
9 总结与展望
9.1 工作总结
本系统面向地瓜智慧医疗车大赛，基于 RDK X5 与 ROS 2 构建了端-边-云协同的自主作业平台。感知层面，提出多级级联识别与云边多模态熔断机制；控制层面，基于 Madgwick 算法与 EKF 融合，实现了 50 Hz 高频位姿输出与高精度轨迹跟踪；系统层面，通过 CPU 绑核与回调隔离，保证了多任务并发的确定性。实测任务成功率超 70%，圆满达成设计目标。
9.2 未来展望
围绕大赛进阶与真实医疗需求，后续将深化以下方面：引入轻量 Transformer 检测器提升极端光照鲁棒性；结合 RAG 检索增强技术融合本地医学知识库；探索基于世界模型或强化学习的端到端避障导航；研究 V2X 任务分配以支持多车协同配送场景。
